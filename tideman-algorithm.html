Overview

This project is my solution to the Tideman algorithm problem from CS50. It is not just a small exercise — it is a non-trivial algorithmic problem involving ranked-choice voting logic, graph-style reasoning, and careful handling of constraints.

Important Technical Ideas

  • Ranked pairs style voting logic
  • Working with preferences
  • Pair comparison
  • Sorting by strength
  • Locking edges carefully
  • Preventing cycles
  • Graph-based reasoning
  • Extracting the correct winner from a constrained structure

What this project demonstrates

  • Algorithmic thinking
  • Graph problem-solving
  • Careful logic design
  • Breaking a difficult problem into smaller phases
  • Handling edge cases and constraints precisely

Things Learned

  • How to think through graph-related logic step by step
  • How to prevent invalid states such as cycles
  • How complex logic becomes manageable when broken into stages
  • How algorithm problems sharpen structured reasoning
